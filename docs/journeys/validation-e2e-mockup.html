Canonical JUnit viewer + one optional field. The body of this dialog is the same JUnit viewer that ships everywhere: failures triaged on top, + N passed browse-by-file underneath, per-test stdout / stderr / traceback / actions. Generic JUnit consumers (e.g. tixmeup) see exactly that — nothing else. The blue-bordered Linked issue panel under the failed test is the one orchestrator-specific extension: it renders only when the test case carries a linked_issue field, which our parser sets when an E2E test drove the orchestrator on an issue. Generic parsers don't set it → tixmeup never sees the panel. No plugin registry yet, no marker protocol — just one optional case field. When we want a second extension, that's when we extract a renderer hook.
