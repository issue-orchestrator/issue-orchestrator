# Documentation map

This repository has three kinds of documentation:

- `docs/public/` — **How to use** Issue Orchestrator (installation, quickstart, configuration).
- `docs/design/` — **Why it’s built this way** (invariants, tradeoffs, contracts). Public, but not a usage guide.
- `docs/internal/` — **How to implement/change it** (agent + contributor notes, task lists, test checklists).

If you're an AI agent implementing changes, start in:
- `docs/internal/IMPLEMENTATION_NOTES.md`
- `docs/internal/planning/IMPLEMENTATION_TASK_LIST.md`
- `docs/internal/planning/TEST_CHECKLIST.md`
